🚀 飞船手动控制消息机制说明文档

详细说明飞船手动控制系统的消息发送机制和消息内容

📋 概述

飞船手动控制系统通过Socket服务向所有连接的客户端广播控制命令。系统支持两种通信方式：原生Socket连接（端口8888）和WebSocket连接（端口8889）。

消息发送机制：
  • 所有控制命令通过 socketService.broadcastMessage(command) 方法广播
  • 消息同时发送到所有连接的Socket客户端和WebSocket客户端
  • 消息格式为纯文本字符串，使用UTF-8编码
  • 系统每30秒自动发送Ping消息（__PING__）检测连接状态

🎮 基本控制命令

基本控制命令包括前进、后退、左转、右转和推进。每个命令在按下时发送开始消息，释放时发送结束消息。

⚡ 跃迁控制命令

跃迁系统是一个多阶段的过程，包括充能、执行和完成三个阶段。跃迁充能需要一定时间，充能完成后会自动执行跃迁。

跃迁流程

按下回车键或长按跃迁按钮 → 开始充能
充能过程中（约2.5秒）→ 充能进度更新
充能达到100% → 充能完成
自动执行跃迁（约6秒）→ 根据充能程度发送不同命令
跃迁执行完成 → 发送完成消息

1. 开始跃迁充能

操作方式

按下 回车 键或长按"跃迁"按钮

条件：当前未在充能、未充能完成且未在执行跃迁

发送的消息

START_JUMP_CHARGE
跃迁命令

说明

  • 开始跃迁充能过程
  • 充能时间约2.5秒（每50毫秒增加4%充能值）
  • 充能过程中可以取消

2. 跃迁充能完成

触发条件

充能值达到100%（自动触发）

发送的消息

JUMP_CHARGE_COMPLETE
跃迁命令

说明

  • 充能完成后会自动开始执行跃迁
  • 此时会显示充能完成提示和粒子效果
  • 无法取消，必须等待跃迁执行完成

3. 执行跃迁

触发条件

充能完成后自动执行（约6秒动画过程）

发送的消息（根据充能程度）

EXECUTE_JUMP_FULL
跃迁命令

条件：充能程度 ≥ 95%

EXECUTE_JUMP_MEDIUM
跃迁命令

条件：充能程度 ≥ 50% 且 < 95%

EXECUTE_JUMP_SHORT
跃迁命令

条件：充能程度 < 50%

注意：正常情况下充能会达到100%，所以通常会发送 EXECUTE_JUMP_FULL 消息。

4. 跃迁完成

触发条件

跃迁执行动画完成（约6秒后）

发送的消息

JUMP_COMPLETE
跃迁命令

说明

  • 跃迁过程完全结束
  • 所有跃迁相关状态重置
  • 可以开始下一次跃迁

5. 取消跃迁

操作方式

在充能过程中按下 回车 键或点击跃迁按钮

条件：正在充能且未开始执行跃迁

发送的消息

CANCEL_JUMP
跃迁命令
重要：一旦开始执行跃迁（充能完成），就无法取消，必须等待跃迁完成。

🔧 消息发送机制详解

消息发送流程

用户操作（按键/点击）
调用 _sendCommand(command) 方法
通过 socketService.broadcastMessage(command) 广播消息
消息发送到所有Socket客户端（端口8888）
消息发送到所有WebSocket客户端（端口8889）
同时通过事件总线发送到本地组件

消息格式

  • 编码：UTF-8
  • 格式：纯文本字符串
  • 大小写：全部大写
  • 分隔符：使用下划线（_）分隔单词

消息分类

消息类型 前缀 示例 说明
开始命令 START_ START_FORWARD 控制动作开始
结束命令 END_ END_FORWARD 控制动作结束
跃迁命令 无固定前缀 START_JUMP_CHARGE
JUMP_CHARGE_COMPLETE
EXECUTE_JUMP_FULL		 跃迁相关操作

⚙️ 技术实现细节

Socket服务配置

  • 原生Socket端口：8888
  • WebSocket端口：8889
  • WebSocket路径：/ws
  • 绑定地址：0.0.0.0（所有网络接口）
  • Ping间隔：30秒
  • Ping消息：__PING__（不记录日志）

消息处理逻辑

消息发送方法

void _sendCommand(String command) { // 1. 通过Socket服务广播消息 socketService.broadcastMessage(command); // 2. 通过事件总线发送到本地组件 if (command.startsWith('START_')) { shipControlEventBus.emitCommand( command.substring(6), isStart: true ); } else if (command.startsWith('END_')) { shipControlEventBus.emitCommand( command.substring(4), isStart: false ); } else { // 跃迁相关命令 shipControlEventBus.emitJumpCommand(command); } }

消息广播机制

广播方法

void broadcastMessage(String message) { // 1. 广播到所有Socket客户端 for (var socket in sockets) { socket.add(utf8.encode(message)); } // 2. 广播到所有WebSocket客户端 for (var webSocket in webSockets) { webSocket.add(message); } }
优势：
  • 支持多客户端同时连接
  • 实时同步所有客户端的控制状态
  • 自动检测和清理断开的连接
  • 支持Socket和WebSocket两种协议

⚠️ 注意事项

消息发送时机

  • 基本控制命令（前进、后退、左转、右转、推进）在按键按下和释放时分别发送开始和结束消息
  • 跃迁充能开始需要用户主动触发，但充能完成和执行是自动的
  • 一旦跃迁开始执行，无法取消，必须等待完成

消息接收处理

  • 客户端应该正确处理所有消息类型，包括未知消息
  • 建议实现消息队列机制，避免消息丢失
  • 注意处理网络延迟和消息乱序的情况
  • Ping消息（__PING__）应该被忽略，不进行业务处理

最佳实践

  • 客户端应该维护当前控制状态，根据START和END消息更新状态
  • 对于跃迁命令，应该按照流程顺序处理，避免状态混乱
  • 建议实现消息确认机制，确保重要消息被正确接收
  • 定期检查连接状态，及时重连断开的连接
