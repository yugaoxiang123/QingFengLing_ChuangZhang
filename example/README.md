# QF控制器 WebSocket测试客户端

这是一个简单的HTML+JavaScript测试客户端，用于测试QF控制器的Socket服务功能。

## 功能特点

- 连接到QF控制器的Socket服务器
- 发送和接收文本消息
- 发送充气命令（格式：`INFLATE:压力值`）
- 实时显示连接状态和消息记录

## 使用方法

1. 确保QF控制器Flutter应用已经启动并运行服务器
2. 在浏览器中打开`index.html`文件
3. 输入服务器IP地址和端口（默认为`127.0.0.1:8888`）
4. 点击"连接服务器"按钮
5. 连接成功后，你可以发送文本消息和充气命令

## 注意事项

### 关于WebSocket

这个测试客户端使用的是WebSocket协议，而不是原生Socket。如果QF控制器使用的是原生Socket而不支持WebSocket协议，则需要进行适配或使用其他测试工具。

原生Socket和WebSocket的主要区别：
- WebSocket是基于HTTP的协议升级，在Web浏览器中可用
- 原生Socket是更底层的TCP/IP连接

### 可能的解决方案

如果测试失败，可以考虑以下方案：

1. 在QF控制器应用中添加WebSocket支持
2. 使用代理服务器转发WebSocket到原生Socket
3. 使用其他非浏览器环境的测试工具（如Postman、Socket.IO客户端等）

## 常见问题排查

- **无法连接到服务器**：
  - 确认QF控制器应用已启动并正常运行
  - 确认服务器IP和端口正确
  - 检查是否存在网络限制或防火墙阻止
  - 确认服务器支持WebSocket协议

- **消息发送失败**：
  - 检查连接是否仍然保持
  - 确认消息格式是否正确
